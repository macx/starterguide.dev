  Wenn du bisher noch keine Berührungspunkte mit einem Terminal hattest, keine
  Sorge! So kannst du prüfen, ob Node.js und die Paketmanager korrekt
  installiert sind.
</Lead>

Öffne unter macOS das Terminal bzw. unter Windows die PowerShell. Bei der Eingabe der folgenden Befehle erhältst du entweder eine Versionsnummer, was bedeutet, dass die Software korrekt installiert ist, oder eine Fehlermeldung, wenn sie nicht installiert ist.

```shell
# Überprüfung von Node.js
node -v

# Überprüfung von npm
npm -v

# Überprüfung von Corepack
corepack -v

# Überprüfung von Homebrew (macOS)
brew -v

# Überprüfung von Chocolatey (Windows)
choco -v
```

## Tipps

- **Du erhältst den Fehler `command not found`**

  Bevor du eine Fehleranalyse startest, starte dein Terminal oder die PowerShell neu. Manchmal werden Umgebungsvariablen erst nach einem Neustart aktualisiert. Die Umgebungsvariablen helfen dem System dabei, die installierten Programme zu finden.

- **Du hattest zuvor schon Paketmanger installiert?**

  Wenn du mit verschiedenen Projekten arbeitest, solltest du darauf achten, nicht mehrere Paketmanager (npm, yarn, pnpm) global installiert zu haben. Für eine saubere Verwaltung empfiehlt sich die Nutzung von Corepack. Mehr dazu findest du im [Corepack-Tutorial](/guides/corepack).

Weitere mögliche Fehler und deren Lösungen findest du auf der Seite [Fehler beheben](/guides/troubleshooting).
