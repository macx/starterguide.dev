## Installation überprüfen

Öffne unter macOS das Terminal und unter Windows die PowerShell. Gib dort die folgenden Befehle ein, um die Installation von Git, Node.js, Yarn und weiteren Tools zu überprüfen.

:::note
In den folgenden Abschnitten kennzeichnen wir die Befehle mit einem vorangestellten `$`, um auch die Ausgabe entsprechend darstellen zu können. Die Eingabe der Befehle erfolgt ohne das `$`.
:::

```shell
# Überprüfung von Node.js
$ node -v

# Überprüfung von npm
$ npm -v

# Überprüfung von Yarn
$ yarn -v

# Überprüfung von Corepack
$ corepack -v

# Überprüfung von Git
$ git -v

# Überprüfung von Homebrew (macOS)
$ brew -v

# Überprüfung von Chocolatey (Windows)
$ choco -v
```

Erhältst du keine Versionsnummer, ist die Software nicht korrekt installiert. In diesem Fall solltest du die Installation wiederholen.

:::tip
Wenn du gerade Node.js, Git oder etwas anderes installiert hast, musst du das Terminal schließen und erneut öffnen, damit die Änderungen wirksam werden. Das gilt sowohl für macOS als auch für Windows.
:::

## Hinweise zu Paketmanager-Versionen

Wenn du mit verschiedenen Projekten arbeitest, solltest du darauf achten, nicht mehrere Paketmanager (npm, yarn, pnpm) global installiert zu haben. Für eine saubere Verwaltung empfiehlt sich die Nutzung von Corepack. Mehr dazu findest du im [Corepack-Tutorial](/guides/corepack).

## Fehler beim Anlegen einer Kopie

Wenn du Beispieldateien von deinem Dozenten oder aus einem anderen Projekt kopierst (z. B. für Übungen mit Parcel, Astro oder anderen Tools), solltest du ausschließlich das `src`-Verzeichnis und die `package.json` übernehmen. Kopiere niemals die Verzeichnisse `node_modules`, `.parcel-cache`, `.yarn`, oder Dateien wie `package-lock.json`, `yarn.lock`, `pnpm-lock.yaml` mit.

**Vorgehen:**

1.  Kopiere das `src`-Verzeichnis und die `package.json` in dein neues Projektverzeichnis.
2.  Lösche alle Lock-Dateien (`package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`) und Caches (`node_modules`, `.parcel-cache`, `.yarn`).
3.  Installiere die Abhängigkeiten neu:

```shell
npm install
# oder
yarn install
# oder
pnpm install
```

Dadurch werden alle benötigten Pakete passend zu deiner Umgebung frisch installiert und du vermeidest Versionskonflikte oder Fehler durch alte Abhängigkeiten.
